### Description

```text
rtl/          可综合 Verilog 源码 / synthesizable Verilog
tb/           testbench，只用于仿真 / simulation-only testbenches
filelist/     源文件列表，由 make init 生成 / file lists generated by make init
constraints/  管脚、时钟、IO 标准约束 / board constraints
docs/         引脚和设计说明 / pin and design notes
build/        自动生成文件，不进 git / generated build artifacts
```

## Simulation Flow

Generate missing testbench templates for every RTL module:

```sh
make sim-init
```

List modules that can get generated testbenches:

```sh
make sim-list
```

Run one testbench and generate a VCD waveform:

```sh
make sim MODULE=board_button
```

Open the waveform with GTKWave:

```sh
make wave MODULE=board_button
```

Run every existing `tb/*_tb.v` testbench:

```sh
make sim-all
```

Generated simulation outputs are written under `sim/` and are ignored by git.

`make flash` uses `--unprotect-flash --skip-reset` by default. The unprotect step
helps when the SPI flash refuses erase/write commands, while skip-reset avoids a
known failure mode where Gowin flash programming finishes successfully and then
hangs during the loader's final reset/SRAM erase step. Use `make flash-reset`
only when you explicitly want the loader's default post-flash reset flow.

If `openFPGALoader` repeatedly reports USB or FTDI errors, unplug and reconnect
the board. If the whole USB stack looks stuck, rebooting the machine may be the
fastest way to reset the host controller and FTDI state.

## Naming Principle

- 模块名：小写字母，单词间用下划线分隔，如 `uart_tx.v`
- 端口名：小写字母，单词间用下划线分隔，如 `clk`, `rst_n`, `data_in`
- 变量名：小写字母，单词间用下划线分隔，如 `counter`, `state`, `data_buffer`
- 参数名：驼峰+下划线+大写开头: 如 `BaudRate`, `DataWidth`
- 文件名：模块名与文件名一致，如 `uart_tx.v` 包含 `uart_tx` 模块
- 后缀：
  - `_i` input
  - `_o` output
  - `_io` inout
  - `_n` active low，低有效
