visibility means important functions and system status should be visible to users, so that users can understand what actions are available and what to do next;

*** Constraints:

**** Internal Consistency:
Internal consistency means operations behave the same withing an application
- e.g, in the same app, all the return button has the same position. all the list item will enter detail page if clicked. all the error inform are in the same style;
- template: Internal consistency refers to desgining operations and interface element to behave the same within the same application or system;
**** External Consistency:
External consistency refers to consistency across different applications, device or real-world conventions. It allows users to transfer prior from one system to another;
- e.g. user are common with icon's meaning in the real work: the bin icon menas bin or delete operation. the mail icon means msg box.

* Alternative designs
Alternative designs are different possible solutions to the same design problem. Considering alternatives helps designers avoid sticking to the first workable idea and identify better designs.
** Divergenet Thinking
Divergent thinking expands the design space by generating many different ideas without immediate judgment. 
** Convergent Thinking
Convergent thinking then evaluates, combines, and eliminates alternatives based on design principles and user needs to select a better solution.

** Method for Generating Alternatives:
- BrainStorm:
  Brainstorming is an unstructured rapid ideation method. Criticism is deferred, and quantity is encouraged over immediate feasibility to generate creative ideas.
- CrazyEight:
  Crazy Eights is a design sprint method where participants sketch eight different design alternatives in eight minutes. It forces designers to move beyond their first obvious idea.
- Morphological Analysis:
  Morphological analysis breaks a product into essential attributes and recombines them in different ways to generate unexpected alternative concepts.
- Balancing constraints nad trade-offs
  When choosing among alternatives, designers need to balance constraints and trade-offs such as cost, time, technical feasibility, usability, accessibility, and user needs.
** Evaluating the Alternative:
*** Heruristic Evaluation
Heuristic evaluation means designers evaluate alternatives against established design principles, such as visibility, feedback, and consistency, to eliminate objectively poor designs before user testing.
eValuating using the principles like 
- Visibility, Feedback, Consistency, Constraints, Affrodance
*** A/B Testing:
A/B testing deploys two alternative designs to different user groups and uses quantitative data, such as click-through rate or task completion rate, to choose the better design.
*** Qualitative User Feedback
involves showing prototyps to user and asking why they prefer one alternatives over another, It provides deeper insight that quatitiative data along cannot explain
