** 5. 
The output from the following code is
#+begin_src java
public class Main {
    public static void main(String[] args) {
        ArrayList<String> list = new ArrayList<String>();
        list.add("New York");
        ArrayList<String> list1 = (ArrayList<String>) (list.clone());
        list.add("Atlanta");
        list1.add("Dallas");
        System.out.println(list1);
    }
}
#+end_src
*** note:
clone() 做的是：
1. 创建一个新的 ArrayList 对象（新地址）
2. 把旧列表里的每个元素引用复制到新列表里
clone 前：
list     → [0x01("New York")]   // ArrayList 对象在 0xA
list1    = null
clone 后：
list     → [0x01("New York")]   // ArrayList 对象在 0xA
list1    → [0x01("New York")]   // ArrayList 对象在 0xB（全新！）
所以
- list.add("Atlanta") → 改了 list 这个 ArrayList 对象的结构，list1 是另一个 ArrayList，不受影响。
- 共享的只是里面的 "New York" 字符串引用，但因为 String 不可变，修改不了。
浅拷贝什么时候会"互相影响"？
如果存的是可变对象（比如 ArrayList<Date>），修改元素内部状态就会影响双方：
list1: [0x02(Date)]           \
list:  [0x02(Date), 0x03, ...] /  共享同一个 Date
list.get(0).setTime(0);   // 修改 Date 对象本身
list1.get(0).getTime();    // 也变了！
一句话：浅拷贝 = 新列表新结构，但元素引用共享。增删列表元素互不影响，修改元素对象本身才会互相影响。
*** sol:
[New York, Dallas]

** 30.
In hash tables, __  probing resolves collisions by sequentially checking the next available slots, while _ probing resolves collisions by checking slots at increasing squared distances from the original hash.
*** note:
- Linear probing（线性探测）
  - It resolves collisions by checking the next slots sequentially.
  - Probe sequence:
    h(key), h(key)+1, h(key)+2, h(key)+3, ...
  - Formula:
    index_i = (h(key) + i) mod table.length

- Quadratic probing（二次探测 / 平方探测）
  - It resolves collisions by checking slots at squared distances from the original hash position.
  - Probe sequence:
    h(key), h(key)+1^2, h(key)+2^2, h(key)+3^2, ...
  - Formula:
    index_i = (h(key) + i^2) mod table.length

- Probing（探测）
  - When a collision occurs, probing means searching for another available slot in the hash table.
*** sol:
Linear Probing,
Quadratic Probing
